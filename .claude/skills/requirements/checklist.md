# 品質条件の標準質問リスト

P2 で走査する標準観点。該当する観点だけを測定可能な基準（QC）として書き、該当しない観点は「他は対象外」の 1 行に畳む。プロダクトごとの上書きは playbook の preset で行う（`presets/` 配下にプロダクト用ファイルがあればそちらを優先する）。

## 標準 5 観点

1. **テスト範囲・レベル**: どの振る舞いをどのレベル（unit / integration / E2E）で検証するか。既存テストへの影響はあるか
2. **性能**: 応答時間・スループット・データ量に影響するか。影響するなら許容基準は何か
3. **互換性**: API・データ・URL・既存ユーザーの操作に対する後方互換は必要か。マイグレーションは必要か
4. **セキュリティ**: 権限チェック、入力検証、個人情報の扱いに変更はあるか
5. **観測性**: 障害時に気づける手段（ログ・メトリクス・アラート）は必要か

## タイプ別の焦点

- **feature**: 5 観点すべてを走査。UI 変更時は P2.5 の合意（状態・操作感）を AC に反映済みか
- **bugfix / hotfix**: 再現条件が AC になっているか（再現テスト先行の入力になる）。回帰範囲が QC として明示されているか
- **refactoring**: 不変条件（守るべき外部振る舞い）が AC の中心になっているか。「既存テスト green 維持」だけで足りるか、特性テストの追加が必要か
- **chore**: 完了の機械的確認方法が QC になっているか。5 観点の走査は省略してよい
- **spike**: 答えるべき問いと time-box が定義されているか。QC は原則不要

## アンチパターン

- 全観点に「影響なし」を並べるチェックリスト演劇（該当しない観点は 1 行に畳む）
- 測定できない基準（「十分速い」「安全である」）
- テストレベルの比率を目標にする（比率は結果であって目標ではない）
