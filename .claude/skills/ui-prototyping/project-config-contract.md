# Project Config Contract

`ui-prototyping` は project ごとの差異を `project-config.md` で受け取る。
最低限以下のセクションを持つこと。

## 必須項目

### 1. プロジェクト概要

- プロジェクト名
- stack
- スタイリング方式
- プレビュー方式

### 2. Stack

現時点では `react-storybook` を明記する。
新しい stack を追加する場合は、対応する variant を `variants/` に追加する。

### 3. パス設定

- 既存 UI コンポーネントの参照先
- デザイントークンの参照先
- プロトタイプ出力先
- ガイドラインや参考実装の参照先

### 4. プレビュー方法

- 起動コマンド
- 表示対象のルール
- プレビュー URL や確認手順

### 5. コーディング規約

- コンポーネント定義パターン
- スタイル定義パターン
- 命名規約
- 禁止パターン

## 推奨項目

- デザイントークン一覧
- 既存コンポーネント参照方法
- アナリティクスや E2E 属性の扱い
- 実装ハンドオフ時に必須の制約

## 責務分離

- variant:
  生成ファイル形式、プレビュー手段、状態表現、実装ハンドオフの型
- preset:
  プロジェクト固有のパス、ルール、トークン、コマンド

技術 stack 自体が違う場合は preset 追加だけではなく variant 追加を検討する。
