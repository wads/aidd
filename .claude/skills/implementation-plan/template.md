# 実装方針テンプレート（draft PR 説明文）

```md
## 実装方針（P5）

参照: {Issue 番号}、設計書 {リンク or なし}、ADR {リンク or なし}

### ステップ

| # | 内容 | 区分 | 対応 AC | 備考 |
|---|---|---|---|---|
| 1 | {再現テストを書く} | TDD | AC-1 | |
| 2 | {設定値を追加する} | DIRECT | QC-2 | 実施手順: {…} / 検証手順: {…} |

### 触らない範囲
- {範囲}: {理由}

### リスクと検証方針
- {リスク}: {検知・回避方法}

### 人間確認ポイント
- ステップ {n} 完了時: {確認内容}
```

ルール:

- DIRECT のステップは実施手順と検証手順を必ず書く（テストが無い分、検証を明文化する）
- bugfix の簡易計画はこの表を 1 段落に圧縮してよい（再現テスト → 修正 → 回帰確認）
- 計画からの逸脱（スコープ変更・想定外・新しい技術判断）が生じたら停止して人間へ確認する
