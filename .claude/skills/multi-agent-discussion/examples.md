# Examples

## 単体利用

```text
/multi-agent-discussion エラーハンドリングの方針。独自エラー型を定義すべきか、標準のエラーで十分か。
```

```text
/multi-agent-discussion 検索のキャッシュ層を Redis にするか、アプリ内メモリにするか、入れないか。
perspectives: Redis 推進 / メモリ推進 / キャッシュなし推進
```

## P3（adr）から呼ばれる場合

`adr` skill が技術判断の選択肢を整理する際、選択肢が割れる・影響が大きいと判断したときに内部で呼ぶ。出力（各案・差分・トレードオフ）は ADR の「検討した選択肢」「決定理由」「トレードオフ」に転記される。

## アンカリング回避が効く例

依頼者が「Redis でキャッシュしたい」と言っている場合でも、各エージェントに Redis / メモリ / キャッシュなしを割り当てて独立調査させる。結果として「そもそもキャッシュ不要（クエリ最適化で足りる）」が最善と分かることがある。単一視点では出てこない結論。
