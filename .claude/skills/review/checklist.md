# Review Checklist

## 規約

### Python

- フォーマッター準拠
- 型ヒント、docstring が既存スタイルと一致
- `SELECT *` を使っていない
- 存在チェックに `COUNT(*)` ではなく `LIMIT 1` を使う
- SQL がインデックスを考慮している
- テストが提供されている

### JavaScript / TypeScript

- フォーマッター、リンター準拠
- 型アノテーションが既存スタイルと一致
- テストが提供されている

### 共通

- 認証情報、シークレットがコードにない
- 変更範囲がタスクスコープに収まっている
- 過剰な抽象化がない
- エラーハンドリングが適切

## 設計

- ビジネスロジック層にシステム都合が混入していない
- データアクセス層にビジネスロジックが混入していない
- 表示、ルーティング層にビジネスロジックが混入していない
- 単一責任を守っている
- 依存差し替えが可能
- 命名が責務を正確に表す
- 既存パターンや命名規則と一貫している

## 仕様適合

- 完了条件を満たしている
- 実装範囲の逸脱がない
- エッジケースを考慮している
- 既存挙動との互換性がある
- テストケースが仕様を網羅している
