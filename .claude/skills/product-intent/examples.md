# Examples

## タイプ確定の例

```text
この依頼は「検索画面の表示崩れを直したい」なので bugfix と判断します。
- タイプ: bugfix（`type-bugfix` / 課題タイプ Bug）
- 理由: 既存の期待挙動からの逸脱の修正で、新しい振る舞いの追加ではないため
よろしければ bugfix 用の軽量テンプレートで進めます。
```

## ヒアリングの冒頭

```text
Product Intent を整理して Jira チケット案を作成します。
背景・意図、ユーザーストーリー、作るもの、対象範囲、受入れ条件、前提・制約、関連情報を教えてください。
まだ決まっていない項目は「未定」で構いません。
```

## 不足確認の例

```text
受入れ条件がまだ完了判断に使える粒度ではありません。
以下を確認させてください。

1. ユーザーがどの操作をしたときの挙動ですか？
2. 正常時に画面上で何が確認できれば完了ですか？
3. 保存や取得に失敗した場合の表示は今回の対象ですか？
```

## 過剰記述を整理する例

```text
API や DB の詳細設計がチケット本文に多く含まれています。
Jira チケットでは Product Intent を中心にし、実装方式は後続の設計または実装方針で扱う方がよいです。
チケット本文には、ユーザーに見える振る舞いと受入れ条件として残します。
```

## チケット例

```md
## 背景・意図
商品を探すユーザーが、毎回同じ検索条件を入力し直している。
再訪時にも前回の条件から探し始められるようにし、商品探索の手間を減らしたい。

## ユーザーストーリー
商品を探しているユーザーとして、
検索条件を保存したい。
なぜなら、次回アクセス時に同じ条件で商品を探し直せるようにしたいから。

## 作るもの
検索画面でユーザーが指定した検索条件を保存し、次回アクセス時に保存済み条件を復元する。

## 対象範囲
- 対象画面・導線: 商品検索画面
- 対象ユーザー・ケース: ログイン済みユーザーが検索条件を指定して保存するケース
- 対象外: 未ログインユーザー、複数条件セットの保存

## 受入れ条件
- ログイン済みユーザーが検索条件を保存したとき、次回アクセス時にも同じ条件が復元される
- 保存済み条件がないユーザーには、初期状態の検索画面が表示される
- 保存に失敗した場合、ユーザーにエラーメッセージが表示される

## 前提・制約
未ログインユーザーは今回の対象外。

## 関連情報
特になし
```

## 受入れ条件の例

- ユーザーが検索条件を保存したとき、次回アクセス時にも同じ条件が復元される
- 未認証ユーザーが対象画面にアクセスしたとき、ログイン画面へリダイレクトされる
