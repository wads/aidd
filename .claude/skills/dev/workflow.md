# Dev Workflow（P1 着手）

## 開始手順

1. チケットの `type-*` ラベルを確認する。ラベルがなければ着手せず、`product-intent` での整備へ差し戻す
2. `workspace-hygiene` の手順で作業ツリーを確認し、未コミット差分を解消する
3. 差分ゼロを確認したら topic branch を作成する（`feat/{チケット番号}-short-desc`）
4. チケットを読解し、自分の言葉で要求を言い直して人間に確認する
5. `routes.md` からルートを特定して提示する。feature の場合は UI に見える変更を含むかを判断し、P2.5（`ui-prototyping`）の実施要否を提案する
6. 人間がルートと P2.5 実施要否を承認したら、ルートと判断理由を 1〜2 行で Jira コメントへ残し、最初のフェーズ skill へ進む

## ルール

- 差分が残っている状態で実装フローを開始しない
- topic branch 作成前にコード変更を始めない
- タイプの特定・差分の扱いに迷ったら人間に確認する
- 着手後にタイプが誤りと判明したらラベルを修正し、本手順からルートを引き直す
