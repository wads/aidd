# Workspace Hygiene Workflow

作業開始前に、必ず以下の順序で進める。

## 基本フロー

1. `git status --short` で未コミット差分を確認する
2. 差分がある場合は、次のいずれかで解消する
   - 直前までの作業として commit する
   - 後で戻す必要があるなら stash する
   - 不要変更であり、ユーザー承認がある場合のみ削除する
3. 差分がゼロになったことを再確認する
4. topic branch を作成する
5. branch 作成後に初めてコード変更やファイル生成を始める

## 禁止事項

- 差分が残ったまま topic branch を作らない
- topic branch 作成前にコード変更を始めない
- 差分の扱いが不明なまま勝手に stash や削除をしない

## 判断ルール

- 差分の所有者や意図が不明な場合は人間に確認する
- 既存差分を巻き取る commit は避け、必要なら別 commit または stash に分ける
- destructive な削除は明示承認がある場合だけ行う
