# Retrospective Workflow

1. 対象フェーズやタスクを確認する
2. Keep / Problem / Try で整理する
3. 特に以下を重点的に抽出する（Learning Intent）
   - 意図が後工程へ伝わらなかった点（どの Intent がどこで途切れたか）
   - 認知負債になりかけた点（記録すべきだったのにされなかった判断）
   - 短期コンテキストの昇格漏れがなかったか
   - プロセスが重かった点（形骸化した記載、不要だった承認）
4. 振り返り結果を GitHub Issueへコメントする
5. playbook の skill・テンプレートへの具体的な改善を提案し、人間が採用したものは playbook への PR またはIssue にする
