# 設計書テンプレート

ファイル名: `docs/design/{連番}-{チケット番号}-short-title.md`（例: `00007-ECS-12345-saved-search.md`）

```md
# 設計: {タイトル}

- チケット: {チケット番号}
- 関連 ADR: {リンク。なければ「なし」}
- Replaces: {置き換える旧設計書。なければ「なし」}

## 目的 / 非目的

{この設計で達成すること。意図的にやらないこと}

## 責務分割

{どのコンポーネント/モジュールが何を担当するか。なぜこの分割か} 🔵/🟡

## データと状態

{扱うデータ、状態の持ち方、ライフサイクル} 🔵/🟡

## 主要フロー

{正常系の流れ。図が必要なら簡潔に}

## 例外ケース

{エラー、空、境界、競合時の振る舞い}

## 守るべき振る舞い

{将来の変更者が知らないと壊す不変条件。テストで表現できるものはテスト名を併記}

## AC 対応

| AC-ID | 満たす設計要素 |
|---|---|
| AC-1 | {セクション・コンポーネント} |

## 未確定事項

{残っている未決と、誰がいつ決めるか}
```

ルール:

- 各記述に信頼性マーカー（🔵 根拠あり / 🟡 推定 / 🔴 要確認）を付ける。承認時に 🟡・🔴 が残っていてはならない
- 書くべき内容がない節は「特になし」ではなく節ごと削る
- 旧設計書は書き換えない。変更は新しい設計書 + Replaces で表す
