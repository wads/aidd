# Human Checkpoints

人間による確認・承認が必要なポイントの一覧。モードに応じてチェックポイントが変わる。

## Checkpoint List by Mode

### Lite（デフォルト）

| Checkpoint | Description | Approval Criteria |
|-----------|-------------|-------------------|
| 要求内容承認 | 整理された要求が正しいか | 要求の網羅性、優先度 |
| 実装確認 | 実装結果が正しいか | テスト通過、コード品質 |

### Standard（Lite + 以下）

| Checkpoint | Description | Approval Criteria |
|-----------|-------------|-------------------|
| 要件承認 | 具体的な要件が適切か | 実現可能性、スコープ |
| 技術決定承認 | 技術的な選択が適切か | トレードオフの妥当性 |

### Full（Standard + 以下）

| Checkpoint | Description | Approval Criteria |
|-----------|-------------|-------------------|
| 設計承認 | 実装方針が適切か | 設計の整合性 |

## How Checkpoints Work

1. 各フェーズの成果物を提示
2. 人間が内容を確認
3. 承認 -> 次のフェーズへ進む
4. 差し戻し -> フィードバックを反映して再提示

## Escalation

以下の場合は自動的に人間にエスカレーション:
- マルチAgent調査で明確な優位選択肢がない場合
- リスクの高い技術的決定
- スコープ変更が必要な場合
- 想定外の問題が発生した場合

## Checkpoint Skip

緊急度が高い場合やユーザーが明示的に許可した場合は、チェックポイントをスキップ可能。
ただし、スキップした旨を記録すること。
