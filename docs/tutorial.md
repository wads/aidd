# Tutorial: AIDD で開発を始める

Claude Code を起動した後の具体的な進め方を、ステップバイステップで説明します。

## 前提

- Quick Start の手順でプロジェクトのセットアップが完了していること
- CLAUDE.md の Project Configuration（Test command 等）が設定済みであること

## まずはこれだけ: `/dev` で始める

AIDD の基本は `/dev` コマンドです。作りたいものを伝えるだけで、適切なプロセスで開発が進みます。

```
> /dev TODOアプリのREST APIを作りたい。タスクのCRUD操作ができるようにしたい。
```

デフォルトでは Lite モード（最小プロセス）で動きます。

## Lite モード（デフォルト）の流れ

### Step 1: 要求確認

Claude が要求を整理し、実装スコープと受け入れ基準を提示します。

**あなたがやること:**
- 内容に問題がなければ承認する
- 不足や修正があればフィードバックする

### Step 2: TDD で実装

`/tdd-cycle` が自動的に実行され、3つのフェーズが繰り返されます:

- **Red**: 失敗するテストを作成 → テスト実行で失敗を確認
- **Green**: テストを通す最小限のコードを実装 → テスト成功を確認
- **Refactor**: コード改善 → テストが引き続き通ることを確認

**あなたがやること:**
- 各フェーズの結果を確認して次に進む
- テスト内容や実装方針に違和感があればフィードバックする
- commit タイミングを判断する

### Step 3: 完了と振り返り

実装完了後、「振り返りしますか？」と聞かれます。

- **Yes**: KPT（Keep/Problem/Try）で振り返りを実施。教訓が `tasks/lessons.md` に蓄積される
- **No**: そのまま完了

## より詳細なプロセスが必要な場合

タスクの規模が大きい場合は、モードを指定できます。

### Standard モード

技術選定や複数ファイルにまたがる機能開発に:

```
> /dev 認証機能を追加したい mode:standard
```

Lite に加えて、要件定義・マルチAgent調査・ADR作成が追加されます。

### Full モード

新アーキテクチャ導入や大規模変更に:

```
> /dev データベースをPostgreSQLからDynamoDBに移行したい mode:full
```

Standard に加えて、フォーマルな要求定義・詳細設計・設計レビューが追加されます。

## 技術的な議論が必要なとき

開発中に判断に迷ったら、いつでも `/multi-agent-discussion` で調査できます。

```
> /multi-agent-discussion エラーハンドリングの方針。独自エラー型を定義すべきか、標準のエラーで十分か。
```

複数のAgentが独立に並行調査し、結果を突き合わせて選択肢を整理してくれます。最終判断はあなたが行います。

## Tips

### Skill を使わない自由な開発もOK

Skill はあくまでガイドです。慣れてきたら Skill を使わずに直接指示してもかまいません。

```
> この関数にバリデーションを追加して。テストも書いて。
```

### 教訓を活用する

`tasks/lessons.md` に蓄積された教訓は、Claude が自動的に参照します。
プロジェクトを重ねるほど、開発プロセスが改善されていきます。

### モード選択の目安

| 迷ったら | モード |
|---------|--------|
| 1-2ファイルの変更 | Lite |
| 技術選定が必要 | Standard |
| アーキテクチャ変更 | Full |
